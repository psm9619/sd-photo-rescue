# sd-photo-rescue

[English](README.md) · [한국어](README.ko.md) · **日本語**

**誤ってフォーマットした**（またはマウントできない）SDカードから、写真・動画をターミナルで復元するツールです。Pythonファイル1つ、インストール不要、**読み取り専用で安全**です。

> フォーマットは通常「目次（ファイルテーブル）」を消すだけで、写真データは上書きされるまでカード上に残っています。このツールはカードに**書き込まず読み取るだけ**で、内容からファイルを復元します（ファイルカービング）。

- ✅ **JPEG**、**RAW**（Canon CR2/CR3、Nikon NEF、Sony ARW、Fujifilm RAF、Olympus ORF、Panasonic RW2、Adobe DNG、HEIC）、**動画**（MP4/MOV）を復元
- ✅ EXIFの撮影日で**期間を絞って**復元可能
- ✅ カードには**一切書き込みません** — 読み取り専用で開きます
- ✅ 対話式：ディスクを自動で見つけ、番号を選ぶだけ
- ✅ Python 3 標準ライブラリのみ — インストール不要
- 💻 macOS と Linux に対応（Windows は現状 `--image` でrawイメージからのみ）

---

## ⚠️ まず最初に

**そのカードの使用を直ちに中止してください。** これ以上撮影しない、カメラやアプリに書き込ませない、再フォーマットしない。1回の書き込みでも、復元可能な写真が永久に上書きされることがあります。カードを抜いてカードリーダーで作業してください。

## クイックスタート

### macOS / Linux

1. ファイル1つ `recover.py` をダウンロード（または本リポジトリを `git clone`）。
   ```bash
   curl -fsSLO https://raw.githubusercontent.com/psm9619/sd-photo-rescue/main/recover.py
   ```
2. カードを挿して実行（rawデバイスの読み取りに `sudo` が必要）：
   ```bash
   sudo python3 recover.py
   ```
3. 質問に答えるだけ：
   - ディスク一覧が出たら → カードの**番号を入力**
   - 復元する種類を選択（写真 / +RAW / +動画）
   - （任意）日付範囲
   - 保存先フォルダを選択

これで完了です。復元ファイルは保存先（既定 `~/recovered`）に入り、`20260612_143022_*.jpg` のように撮影時刻の名前が付くため時系列で並びます。

> **必ず正しいディスクを選んでください。** ツールは容量と「内蔵/外付け」を表示し、システムディスクには警告します。カードは通常、小さい**外付け/リムーバブル**（例：64GB / 128GB）です。

## オプション（繰り返し実行 / 自動化向け）

指定しなかった項目は対話式で尋ねます。

```bash
sudo python3 recover.py \
    --disk /dev/disk4 \          # macOS; Linux は例 /dev/sdb
    --out ~/recovered \
    --types jpeg,raw,video \     # jpeg, raw, video のいずれか
    --date-from 2026-06-12 \     # --date-* を両方省略すると全日付を復元
    --date-to   2026-06-12 \     # または1日だけ: --date 2026-06-12
    --megapixels 26              # 任意: カメラの画素数（フルサイズとサムネを仕分け）

python3 recover.py --list                       # ディスク一覧だけ表示
python3 recover.py --image card.img --out ~/out # デバイスの代わりにイメージファイルから復元
```

イメージファイル（`--image`）での作業には sudo は不要です。お好みで、先に `ddrescue` で読み取り専用イメージを作成し、そこから復元してもかまいません。

### 出力フォルダの構成

- メインフォルダ — 復元したファイル（日付が読めれば撮影時刻の名前）
- `_unknown_date/` — 日付を読めなかったファイル（日付フィルタ指定時のみ）
- `_other_size/` — `--megapixels` に合わないファイル（多くはサムネ/プレビュー）
- `_partial/` — 破損・不完全なカービング（`.partial`）— 開けないことがあります

### カメラRAW対応

正しい拡張子で認識：**Canon** CR2/CR3、**Nikon** NEF、**Sony** ARW、**Fujifilm** RAF、**Olympus/OM** ORF、**Panasonic** RW2、**Adobe/Leica** DNG、**Pentax** PEF、**Samsung** SRW、**HEIC**。多くのRAWは内部的にTIFFベースのため、この一覧にないメーカーでも通常は復元できます — ただし `.tif` として保存されるので、名前を変えてください。一部の非TIFF形式（例：Sigma X3F、Phase One IIQ）は未対応です。

## 仕組み（概要）

1. デバイスを一度走査し、各ファイルの先頭を**シグネチャ**で見つけます（例：JPEGは `FF D8 FF` で始まる）。
2. 各ファイルの本当の**終端**を正しく特定します：
   - JPEG：マーカーセグメントをたどり、埋め込みサムネイルを終端と誤認しない
   - HEIC/CR3/MP4/MOV：ISO-BMFF のボックス構造をたどる
   - TIFFベースのRAW：IFDチェーンを解析して実際のサイズを算出（埋め込みプレビューで切れない）
   - RAF：ヘッダからサイズを読む
3. （任意）**EXIFの撮影日**を読み、指定した期間だけ残します。
4. 復元したファイルを保存先に書き出します。カードは読み取るだけです。

詳細は `recover.py` を参照 — コメント付きで、標準ライブラリのみを使用しています。

## トラブルシューティング

- **ディスクが出ない / 権限エラー** → `sudo` で実行してください。
- **ほとんど/まったく復元できない** → クイックフォーマットではなく、**完全消去（セキュア消去/低レベルフォーマット）** だったか、すでに新しい写真で上書きされた可能性があります。クイックフォーマットは復元可能、完全消去は通常不可。
- **元のファイル名（`DSCF1234.JPG`）が戻らない** → 仕様です。フォーマットで名前テーブルが消えるため、代わりに撮影時刻の名前で保存されます。
- **遅い** → カードを一度すべて読みます。128GB で通常20〜60分、遅い/USB2リーダーや動画ではさらにかかります。進捗表示が更新され続けている限り**固まっていません**。速度はカード/リーダーの読み取り速度で決まり、カードを一度読むことは避けられません。
- **一部のファイルが開けない** → カービングは**断片化した**ファイル（大きな動画/使い込んだカード）を再構成できません。それらは `_partial/` に入るか、わずかに破損することがあります — カービングの通常の限界です。
- **Linux**：`lsblk`（util-linux、既定で同梱）と `sudo` が必要です。

---

## 安全性

このツールはカードを `O_RDONLY`（読み取り専用）で開き、読み取るだけです。書き込み・フォーマット・変更は行いません。復元ファイルは**別の**フォルダに保存します。読み取り前に、macOS ではカードをアンマウントしてOSが書き込めないようにし、Linux ではアンマウントを試み、できない場合は警告します。いずれの場合もカードは厳密に読み取り専用でのみ開きます。
